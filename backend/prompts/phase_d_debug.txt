【結合セル関連のよくあるバグと修正方法】
ファイル構造に「Merged cells」または「テンプレートシート」が含まれる場合、以下を確認すること:

1. 結合セルに書き込む場合は左上セルのみに書いているか？
   - 誤り: `ws['B7'].value = x` （A7:D7 の結合範囲なら B7 は左上でない）
   - 正しい: `ws.cell(7, 1).value = x` または `ws['A7'].value = x`

2. テンプレートコピーに `copy_worksheet` を使っているか？
   - 誤り: `new_wb = openpyxl.Workbook()` でゼロから作成（結合セルが失われる）
   - 正しい: `new_ws = wb.copy_worksheet(template_ws)`

3. 行番号がずれていないか？
   - テンプレートの行番号はファイル構造の「行N:」に記載された番号をそのまま使うこと
   - pandas の DataFrame インデックス（0始まり）と openpyxl の行番号（1始まり）を混同しない

4. 結合セルを壊す操作をしていないか？
   - `ws.merge_cells()` を既存の結合範囲に対して呼ばない
   - 行挿入・削除後に結合範囲が正しく更新されているか確認

5. 別ブックへのシートコピーは正しく行っているか？
   - 誤り: `new_wb['シート名'] = ws`（これは無効な代入でエラーになる）
   - 誤り: `new_wb.copy_worksheet(ws)`（同一ブック内でのみ有効）
   - 正しい: セルをループでコピーする関数を使うこと:
     ```python
     from copy import copy
     def copy_sheet_to_workbook(src_ws, dst_wb, new_name):
         dst_ws = dst_wb.create_sheet(new_name)
         for row in src_ws.iter_rows():
             for cell in row:
                 dst_cell = dst_ws.cell(row=cell.row, column=cell.column, value=cell.value)
                 if cell.has_style:
                     dst_cell.font = copy(cell.font)
                     dst_cell.fill = copy(cell.fill)
                     dst_cell.border = copy(cell.border)
                     dst_cell.alignment = copy(cell.alignment)
                     dst_cell.number_format = cell.number_format
         for merge in src_ws.merged_cells.ranges:
             dst_ws.merge_cells(str(merge))
         return dst_ws
     ```
